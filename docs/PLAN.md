# Guardrails First Landing Page — Implementation Plan

> Frozen copy of the plan agreed at the start of this project (originally at `~/.claude/plans/logo-is-in-the-joyful-duckling.md`). Committed to the repo as a fallback so the agreed scope survives even if local Claude state (plan + memory) is lost on a machine format. Mid-build status updates live in CLAUDE.md, not here.

## Context

Guardrails First is the user's new content brand under the umbrella of Real Agents Academy (`realagents.academy`). It is the show / content surface; future paid courses will live under the umbrella domain. The brand needs a polished single-page landing site at `guardrailsfirst.com` whose only job is to drive newsletter subscriptions to `guardrailsfirst.substack.com` while rendering correctly in social previews (LinkedIn, X, Bluesky, Substack).

The site has to be free to host (SWA free tier), Lighthouse 95+ on mobile, and structured so that adding an essay archive or course funnel later is incremental — not a rewrite. Domain is registered at Namecheap and currently has live M365 and email-forwarding records that **must not break** when web records are added.

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| **Tech stack** | Astro (SSG) | Outputs static HTML/CSS, ships zero JS by default, MDX-ready for the future essay archive. Vanilla would require a rewrite when essays land. |
| **GitHub org** | `realagents-academy` (the bare `realagents` was taken). Repo: `realagents-academy/guardrailsfirst`. | Matches umbrella brand architecture. Future repos all live under one identity. Migrating personal → org later breaks every link. |
| **OG image** | Hand-designed 1200×630 PNG, committed to `public/og-image.png` | One page = one card. Dynamic generation pays off only when there are N essays. Swap to dynamic OG via `@vercel/og` or satori when essays ship — ~30 min job, not a rewrite. |
| **Analytics** | None for v1 | No traffic to measure pre-launch. SWA portal already shows requests/bandwidth. Add Cloudflare Web Analytics when first essay ships. |
| **Social icons in footer (v1)** | Substack, YouTube, email (`hello@guardrailsfirst.com`). Add X if active by build time. Bluesky / LinkedIn / Instagram / Threads / GitHub omitted until live. | Three working icons beats eight with broken links. |
| **CTA copy** | Single button: "Subscribe". Explainer paragraph below carries the "newsletter" context. | Visually cleanest. |
| **Logo asset** | Use existing `pictures/GuardrailsFirstLogo.png` as source. Generate favicon set + apple-touch-icon from it. SVG is a nice-to-have, not blocking. | The PNG is already on-brand (navy bg, teal monogram). For an SVG we'd either auto-trace or hand-rebuild — defer until there's friction. |

## Tech Stack

- **Astro** (latest stable). Output: pure static HTML/CSS, no client JS unless explicitly added.
- **CSS**: hand-written in `src/styles/global.css` with design tokens. No CSS framework. Tailwind would add tooling weight that one page doesn't justify.
- **Fonts**: `Inter` via `@fontsource/inter` (self-hosted, subsetted) — avoids third-party request to Google Fonts. System stack as fallback.
- **No client JS** for v1.

## Visual Design

| Token | Value |
|---|---|
| `--color-bg` | `#0B1F3A` (navy) |
| `--color-accent` | `#5EEAD4` (teal) |
| `--color-text` | `#F8FAFC` (off-white) |
| `--color-text-muted` | `#94A3B8` (slate-400) |
| `--font-sans` | `Inter`, system stack fallback |
| `--max-width` | `64rem` |
| Spacing | `4 / 8 / 16 / 24 / 32 / 48 / 64 / 96 px` |

Layout: centered column, generous vertical rhythm. Hero takes ~70vh with logo top, then name, tagline, subtitle, CTA, explainer. Footer pinned below hero (not sticky). Mobile-first.

Contrast: teal `#5EEAD4` on navy `#0B1F3A` ≈ 10:1 — well above WCAG AA.

Reduced motion: respect `prefers-reduced-motion`. No autoplaying animations.

## DNS Plan (Namecheap)

### Records that already exist — DO NOT TOUCH

| Type | Host | Purpose |
|---|---|---|
| MX | `@` | Namecheap email forwarding (`hello@` → `araujoinnovations.ch`) |
| TXT | `@` | M365 SPF (`v=spf1 include:spf.protection.outlook.com ...`) |
| TXT | `selector1._domainkey` | M365 DKIM |
| TXT | `selector2._domainkey` | M365 DKIM |
| TXT | `_dmarc` | DMARC policy |
| CNAME | `autodiscover` | M365 (if present) |
| TXT | `_github-challenge-realagents-academy` | GitHub org verification |

### New records to add (during steps 13 and 15, with values from SWA)

| Step | Type | Host | Value | TTL |
|---|---|---|---|---|
| 13 | CNAME | `www` | `<swa-app-name>.azurestaticapps.net` | Automatic |
| 15 | TXT | `_dnsauth` | `<token-from-SWA>` | Automatic |
| 15 | A | `@` | `<apex-IP-from-SWA>` | Automatic |

**Apex caveat**: Namecheap parked-domain defaults sometimes include an A record on `@` to a parking page plus URL Redirect rules. Both must be removed before adding the SWA A record. URL forwarding records on `@` will silently override the A record.

**CAA check**: confirm there are no CAA records at Namecheap restricting issuers. Default = none = Let's Encrypt and DigiCert can both issue. Most domains have none; just verify in Advanced DNS panel before step 16.

### Future record (don't add now — reserve mental space)

| Type | Host | Value | When |
|---|---|---|---|
| TXT | `_atproto` | `did=<bluesky-DID>` | When Bluesky account is created and domain-as-handle is wanted |

Sits on a different host label than M365 records — no conflict when added.

## Verification

End-to-end checks before declaring v1 done:

1. **Local dev**: `npm run dev` serves at `http://localhost:4321`. Hero renders with logo, copy, CTA. Footer renders with the live social icons only. Clicking CTA opens Substack subscribe page in a new tab.
2. **Local build**: `npm run build && npm run preview` produces a static `dist/` and serves identically to dev.
3. **Lighthouse (local preview, mobile profile)**: ≥95 on Performance, Accessibility, Best Practices, SEO.
4. **Deploy to SWA**: push to `main` triggers GitHub Action; deploy succeeds; `*.azurestaticapps.net` URL renders identically to local preview.
5. **Custom domain**: `https://guardrailsfirst.com` and `https://www.guardrailsfirst.com` both serve the site over valid TLS. HTTP redirects to HTTPS.
6. **Mail still works**: send a test email to `hello@guardrailsfirst.com` and confirm forwarding arrives. Confirm SPF/DKIM/DMARC pass via headers when sending from M365.
7. **OG previews** validate on Twitter Card Validator, LinkedIn Post Inspector, Meta Sharing Debugger, Substack note preview.
8. **Production Lighthouse** on `https://guardrailsfirst.com` (mobile): ≥95 on all four categories.
9. **Security headers**: `curl -I https://guardrailsfirst.com` shows `Strict-Transport-Security`, `X-Content-Type-Options: nosniff`, `Referrer-Policy`, and a sane CSP.
10. **Future-proofing smoke test**: adding `src/pages/about.astro` builds and routes correctly without config changes — proves the path to `essays/[slug].mdx` is unblocked.
